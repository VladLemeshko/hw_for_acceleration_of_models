# 3. Архитектура модели BERT

## 3.1 Общая структура

BERT (Bidirectional Encoder Representations from Transformers) представляет собой **стек энкодерных блоков Transformer**, предложенных в работе "Attention Is All You Need" (Vaswani et al., 2017). В отличие от оригинального Transformer, BERT использует только encoder-часть архитектуры, что позволяет эффективно обрабатывать входную последовательность целиком с учётом двунаправленного контекста.

### Высокоуровневая схема архитектуры

```
┌─────────────────────────────────────────────────────────────┐
│                    Classification Head                      │
│                    (Linear + Softmax)                       │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                      Pooler Layer                           │
│              (Linear + Tanh, только [CLS])                  │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│              Transformer Encoder Block × 12                 │
│    ┌─────────────────────────────────────────────────────┐  │
│    │  Layer Normalization                                │  │
│    │            ↓                                        │  │
│    │  Multi-Head Self-Attention (12 heads)               │  │
│    │            ↓                                        │  │
│    │  Residual Connection + Dropout                      │  │
│    │            ↓                                        │  │
│    │  Layer Normalization                                │  │
│    │            ↓                                        │  │
│    │  Feed-Forward Network (768 → 3072 → 768)            │  │
│    │            ↓                                        │  │
│    │  Residual Connection + Dropout                      │  │
│    └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                    Embedding Layer                          │
│   Token Embeddings + Position Embeddings + Segment Emb.     │
│                    + Layer Normalization                    │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                       Input Tokens                          │
│           [CLS] token_1 token_2 ... token_n [SEP]           │
└─────────────────────────────────────────────────────────────┘
```

## 3.2 Детальное описание компонентов

### 3.2.1 Embedding Layer

Слой эмбеддингов преобразует входные токены в векторные представления фиксированной размерности.

**Компоненты:**

| Эмбеддинг | Размерность | Назначение |
|-----------|-------------|------------|
| Token Embedding | vocab_size × hidden_dim = 30522 × 768 | Векторное представление токенов |
| Position Embedding | max_seq_len × hidden_dim = 512 × 768 | Кодирование позиции в последовательности |
| Segment Embedding | 2 × hidden_dim = 2 × 768 | Разделение сегментов (предложений) |

**Операции:**
```
embedding = token_emb(input_ids) + position_emb(positions) + segment_emb(segments)
output = LayerNorm(embedding)
output = Dropout(output, p=0.1)
```

**Количество параметров в Embedding Layer:**
- Token: 30522 × 768 = 23,440,896
- Position: 512 × 768 = 393,216
- Segment: 2 × 768 = 1,536
- LayerNorm: 768 × 2 = 1,536
- **Итого: ~23.8 млн параметров (~21.6% от общего числа)**

### 3.2.2 Multi-Head Self-Attention

Механизм внимания позволяет каждому токену «обращать внимание» на все остальные токены в последовательности, взвешивая их вклад в финальное представление.

**Формулы:**

Для каждой головы внимания h ∈ {1, ..., H}:

$$Q_h = XW_h^Q, \quad K_h = XW_h^K, \quad V_h = XW_h^V$$

$$\text{Attention}_h(X) = \text{softmax}\left(\frac{Q_h K_h^T}{\sqrt{d_k}}\right) V_h$$

$$\text{MultiHead}(X) = \text{Concat}(\text{head}_1, ..., \text{head}_H) W^O$$

**Размерности:**

| Параметр | Значение | Описание |
|----------|----------|----------|
| H (число голов) | 12 | Количество параллельных attention-голов |
| d_model (hidden_dim) | 768 | Размерность скрытого состояния |
| d_k = d_v = d_model / H | 64 | Размерность каждой головы |

**Матрицы весов на один слой:**

| Матрица | Размерность | Параметров |
|---------|-------------|------------|
| W^Q | 768 × 768 | 589,824 |
| W^K | 768 × 768 | 589,824 |
| W^V | 768 × 768 | 589,824 |
| W^O | 768 × 768 | 589,824 |
| Biases | 768 × 4 | 3,072 |
| **Итого на слой** | — | **2,362,368** |

### 3.2.3 Feed-Forward Network (FFN)

Каждый transformer-блок содержит полносвязную сеть с одним скрытым слоем и нелинейной активацией GELU.

**Формула:**
$$\text{FFN}(x) = \text{GELU}(xW_1 + b_1)W_2 + b_2$$

**Размерности:**

| Слой | Входная размерность | Выходная размерность | Параметров |
|------|---------------------|----------------------|------------|
| W₁ (up-projection) | 768 | 3072 | 2,359,296 |
| b₁ | — | 3072 | 3,072 |
| W₂ (down-projection) | 3072 | 768 | 2,359,296 |
| b₂ | — | 768 | 768 |
| **Итого на слой** | — | — | **4,722,432** |

**Примечание:** FFN содержит в 2 раза больше параметров, чем attention-блок, что делает его значительным потребителем памяти и вычислительных ресурсов.

### 3.2.4 Layer Normalization

Нормализация применяется до каждого основного блока (pre-norm в некоторых реализациях) или после (post-norm в оригинальном BERT).

**Формула:**
$$\text{LayerNorm}(x) = \gamma \cdot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$$

**Параметры на слой:** γ и β размерности 768, итого 1,536 параметров × 2 (для attention и FFN) = 3,072 на блок.

### 3.2.5 Residual Connections

Остаточные соединения позволяют градиентам беспрепятственно протекать через глубокую сеть:

```
output = LayerNorm(x + Sublayer(x))
```

Residual connections не добавляют параметров, но влияют на паттерны доступа к памяти.

### 3.2.6 Pooler и Classification Head

**Pooler:**
- Извлекает скрытое состояние [CLS]-токена
- Применяет линейное преобразование: 768 → 768
- Активация: Tanh
- Параметров: 768 × 768 + 768 = 590,592

**Classification Head:**
- Линейный слой: 768 → num_classes (50)
- Softmax для получения вероятностей
- Параметров: 768 × 50 + 50 = 38,450

## 3.4 Ключевые размерности

| Обозначение | Значение | Описание |
|-------------|----------|----------|

## 3.5 Наиболее ресурсоёмкие компоненты

### По количеству параметров (влияние на память)

1. **FFN-слои (51.4%)** — содержат более половины всех параметров модели
2. **Attention-слои (25.7%)** — матрицы проекций Q, K, V, O
3. **Embedding-слой (21.6%)** — большой словарь токенов

### По объёму вычислений (FLOPs)

Оценка вычислительной сложности для входа размера B × S:

| Операция | FLOPs | Доля |
|----------|-------|------|
| Attention QKV projection | 6 × B × S × H² × L | ~25% |
| Attention score (Q·K^T) | 2 × B × A × S² × D_head × L | ~15% |
| Attention output (scores·V) | 2 × B × A × S² × D_head × L | ~15% |
| Attention output projection | 2 × B × S × H² × L | ~8% |
| FFN first linear | 2 × B × S × H × I × L | ~18% |
| FFN second linear | 2 × B × S × I × H × L | ~18% |
| Embedding lookup | Незначительно | <1% |

**Итого для B=1, S=128:**
- Attention: ~2.3 GFLOP
- FFN: ~2.9 GFLOP
- **Всего: ~5.2 GFLOP на один forward pass**

### По времени выполнения

На практике распределение времени может отличаться от распределения FLOPs из-за:
- Различной арифметической интенсивности операций
- Накладных расходов на запуск CUDA kernel'ов
- Ограничений пропускной способности памяти

## 3.6 Особенности архитектуры, влияющие на оптимизацию

### Квадратичная сложность attention

Вычислительная сложность self-attention по длине последовательности: **O(S²)**

При увеличении длины последовательности:
- S = 64 → базовый уровень
- S = 128 → 4× больше вычислений в attention
- S = 256 → 16× больше вычислений в attention
- S = 512 → 64× больше вычислений в attention

### Независимость между головами attention

12 голов attention вычисляются независимо, что создаёт возможности для параллелизации, но также увеличивает количество мелких kernel-вызовов.

### Детерминированная структура

В отличие от RNN, все слои BERT имеют фиксированную структуру и не зависят от входных данных, что упрощает статическую оптимизацию графа вычислений.

### Отсутствие авторегрессии

BERT обрабатывает всю последовательность целиком, без необходимости последовательной генерации токенов. Это позволяет максимально эффективно использовать батчинг и параллельные вычисления.
